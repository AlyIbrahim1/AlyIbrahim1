---

### 🏠 [Rent Manager](https://github.com/AlyIbrahim1/RentManagement_app) *(private)*

[![CI](https://github.com/AlyIbrahim1/RentManagement_app/actions/workflows/ci.yml/badge.svg)](https://github.com/AlyIbrahim1/RentManagement_app/actions/workflows/ci.yml)

A full-stack property management app built for real family use — not a portfolio toy. Landlords get a dashboard with live KPIs (revenue, occupancy, collection rate), per-renter payment history, one-click PDF receipt generation, and multi-tenant auth with full data isolation between workspaces.

**Stack:** FastAPI + SQLAlchemy · React 18 + TypeScript + Tailwind · Supabase (PostgreSQL + Auth + Storage) · ReportLab · GitHub Actions CI

The interesting bits: `unpaidMonths` and `rentDue` are computed at query time rather than persisted — so the data is always consistent without a sync job. All protected routes are tenant-scoped and return `404` on cross-tenant access rather than `403`, so you can't even probe what exists. Migrated from a legacy SQLite database without losing a single record.
